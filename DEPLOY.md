# Деплой в Yandex Cloud (Object Storage) + автодеплой через GitHub Actions

Прототип — статический сайт, поэтому хостим в **Object Storage** (бакет со static website hosting), а каждый пуш в `main` автоматически заливается туда через **GitHub Actions**.

Схема: `git push` → GitHub Actions → `aws s3 sync` → бакет → сайт обновился.

---

## Что делаешь ты (один раз)

### 1. Создать бакет
Консоль Yandex Cloud → **Object Storage** → **Создать бакет**:
- **Имя**: например `lecta-smotrelka` (должно быть уникальным во всём YC).
- **Доступ на чтение объектов**: **Публичный**.
- Остальное по умолчанию → создать.

Затем открыть бакет → вкладка **Веб-сайт** (Website) → включить хостинг:
- **Главная страница (index)**: `index.html`
- **Страница ошибки (error)**: `index.html`  ← важно, чтобы работали ссылки на персонажей (`/temka1` и т.п.)
- Сохранить. Там же будет адрес сайта: `http://<бакет>.website.yandexcloud.net`

### 2. Сервисный аккаунт + статические ключи
Консоль → каталог → **Сервисные аккаунты** → создать (например `deployer`), роль **`storage.editor`**.
Затем у этого аккаунта → **Создать ключ → Статический ключ доступа**.
Сохрани **Идентификатор ключа (Key ID)** и **Секретный ключ** — секрет показывается один раз.

### 3. GitHub-репозиторий
Создай пустой репозиторий на GitHub (например `smotrelka`). Локально уже сделан `git init` и первый коммит — осталось подключить remote и запушить:
```bash
cd "Прототип смотрелки 2.0"
git branch -M main
git remote add origin git@github.com:ТВОЙ_ЛОГИН/smotrelka.git   # или https://...
git push -u origin main
```

### 4. Секреты в GitHub
Репозиторий → **Settings → Secrets and variables → Actions → New repository secret**, добавь три:
| Имя | Значение |
|---|---|
| `YC_S3_KEY_ID` | Идентификатор статического ключа |
| `YC_S3_SECRET` | Секретный ключ |
| `YC_BUCKET` | имя бакета (например `lecta-smotrelka`) |

### 5. Готово
После пуша (или вручную: вкладка **Actions → Deploy → Run workflow**) прогон зальёт сайт.
Открой `http://<бакет>.website.yandexcloud.net`.
Дальше **любой пуш в `main` деплоится автоматически.**

## Запасной вариант — ручной деплой с ноутбука
Если не хочешь ждать CI:
```bash
pip3 install awscli        # если ещё не стоит
aws configure              # введи свои Key ID / Secret, регион ru-central1, формат json
YC_BUCKET=lecta-smotrelka ./deploy.sh
```

## HTTPS и красивый домен (по желанию, следующий шаг)
Website-endpoint бакета работает по **HTTP**. Чтобы получить **HTTPS** и свой домен:
- поднять **Yandex Cloud CDN** перед бакетом, привязать домен и бесплатный сертификат Let's Encrypt.
Скажи — распишу шаги и добавлю инвалидацию CDN-кэша в workflow.
